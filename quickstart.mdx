---
title: "Get started with p2panda"
description: "Build and run a local p2p node in Rust that publishes typed messages and subscribes to a topic stream with automatic peer sync in six steps."
---

This guide walks you through spawning a p2panda node, creating a topic, and exchanging typed messages with other peers on the network. By the end you will have a working program that publishes a chat message locally and processes incoming messages from any peer sharing the same topic — all with automatic sync and crash-resilient delivery.

## Prerequisites

- Rust 1.80 or later with a working `cargo` installation
- Familiarity with `async`/`await` and the tokio runtime

## Steps

<Steps>
  <Step title="Add dependencies">
    Add p2panda and its required companions to your `Cargo.toml`. The `serde` and `futures-util` crates are needed for message serialisation and stream iteration respectively.

    ```toml Cargo.toml
    [dependencies]
    p2panda = "0.6.1"
    tokio = { version = "1", features = ["full"] }
    serde = { version = "1", features = ["derive"] }
    futures-util = "0.3"
    ```
  </Step>

  <Step title="Spawn a node">
    `p2panda::spawn()` starts a node with default configuration: an in-memory SQLite database and mDNS discovery so it can find other nodes on the same LAN automatically.

    ```rust src/main.rs
    #[tokio::main]
    async fn main() -> Result<(), Box<dyn std::error::Error>> {
        let node = p2panda::spawn().await?;
        println!("node id: {}", node.id());
        Ok(())
    }
    ```

    The node runs fully in the background. You do not need to poll or tick it manually.
  </Step>

  <Step title="Create a topic">
    A `Topic` is a fixed-length byte array that identifies a shared data space. Any node that uses the same topic bytes will discover each other and sync their data automatically.

    ```rust src/main.rs
    use p2panda::Topic;

    let chat_topic = Topic::random();
    println!("topic id: {}", chat_topic);
    ```

    Share the printed topic identifier with another node to let it join the same conversation.
  </Step>

  <Step title="Open a topic stream">
    `node.stream::<M>(topic)` returns a `(StreamPublisher, StreamSubscription)` pair typed over your message type `M`. p2panda serialises `M` to CBOR automatically — your type just needs to implement `serde::Serialize` and `serde::Deserialize`.

    ```rust src/main.rs
    use serde::{Deserialize, Serialize};

    #[derive(Serialize, Deserialize)]
    struct ChatMessage {
        text: String,
    }

    let (tx, mut rx) = node.stream::<ChatMessage>(chat_topic).await?;
    ```
  </Step>

  <Step title="Publish a message">
    Call `tx.publish(message)` to create and sign an operation locally, persist it, and forward it to connected peers. Publishing works even when no peers are online — the message is stored and gossiped the next time a peer connects.

    ```rust src/main.rs
    tx.publish(ChatMessage {
        text: "Hello, p2panda!".into(),
    })
    .await?;
    ```
  </Step>

  <Step title="Receive and process events">
    Iterate over the subscription stream with `rx.next().await`. The stream emits `StreamEvent` variants for both application messages and network-level events such as sync sessions.

    ```rust src/main.rs
    use futures_util::StreamExt;
    use p2panda::streams::StreamEvent;

    while let Some(event) = rx.next().await {
        // Track sync progress.
        if let StreamEvent::SyncStarted {
            remote_node_id,
            incoming_bytes,
            ..
        } = &event
        {
            println!("syncing {} bytes from {}", incoming_bytes, remote_node_id);
        }

        // Handle application messages.
        if let StreamEvent::Processed { operation, .. } = event {
            println!(
                "[{}] {}: {}",
                operation.timestamp(),
                operation.author(),
                operation.message().text,
            );
        }
    }
    ```
  </Step>
</Steps>

## Full example

The following program combines all steps above into a self-contained chat node. It is modelled after the `todo.rs` example in the p2panda repository, simplified to a single message type.

```rust src/main.rs
use futures_util::StreamExt;
use p2panda::streams::StreamEvent;
use p2panda::Topic;
use serde::{Deserialize, Serialize};

/// A simple chat message replicated across all peers sharing the same topic.
#[derive(Debug, Serialize, Deserialize)]
struct ChatMessage {
    text: String,
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Spawn a node with in-memory storage and mDNS peer discovery.
    let node = p2panda::spawn().await?;

    // Create (or reuse) a topic. Pass a topic id as the first CLI argument to
    // join an existing chat instead of starting a new one.
    let args: Vec<String> = std::env::args().collect();
    let topic: Topic = if args.len() > 1 {
        args[1].parse()?
    } else {
        Topic::random()
    };

    println!("node id : {}", node.id());
    println!("topic id: {}", topic);
    println!("---");

    // Open a typed publish/subscribe stream for this topic.
    let (tx, mut rx) = node.stream::<ChatMessage>(topic).await?;

    // Publish a greeting from this node.
    tx.publish(ChatMessage {
        text: "Hello from this node!".into(),
    })
    .await?;

    // Process events from the network.
    while let Some(event) = rx.next().await {
        if let StreamEvent::SyncStarted {
            remote_node_id,
            incoming_bytes,
            ..
        } = &event
        {
            println!("sync started with {remote_node_id}, {} bytes incoming", incoming_bytes);
        }

        if let StreamEvent::Processed { operation, .. } = event {
            println!(
                "[{}] {}: {}",
                operation.timestamp(),
                operation.author(),
                operation.message().text,
            );
        }
    }

    Ok(())
}
```

<Note>
By default p2panda uses mDNS discovery, which works on local area networks only. To connect peers over the Internet you need to configure a bootstrap node and a relay server. See the [node setup guide](/guides/node-setup) for details.
</Note>
