## What p2panda provides

p2panda is built around a high-level **Node API** that orchestrates all the pieces — networking, discovery, sync, event streaming, causal ordering, and storage — in one ergonomic Rust API. For lower-level use cases, each crate can be used independently.

## Key characteristics

- **Broadcast-only by design** — data is compatible with post-internet communication infrastructure like packet radio, LoRa, BLE, or a USB stick
- **Transport-agnostic** — today runs over QUIC/iroh; designed to support mesh and radio transports in the future
- **CRDT-compatible** — operation logs work with any CRDT or application data type
- **Fork-tolerant** — single-writer logs with pruning and multi-writer causal ordering
- **Crash-resilient** — at-least-once delivery with acknowledgements and replay

<Warning>
p2panda is under active development. APIs are not yet considered stable for production use. Breaking changes may occur before v1.0.0.
</Warning>

## Installation

Add the high-level Node API to your Rust project:

```toml Cargo.toml
[dependencies]
p2panda = "0.6.1"
tokio = { version = "1", features = ["full"] }
serde = { version = "1", features = ["derive"] }
```

Or pick individual crates for lower-level use:

```toml Cargo.toml
[dependencies]
p2panda-core = "0.6.1"
p2panda-net = "0.6.1"
p2panda-sync = "0.6.1"
p2panda-encryption = "0.6.1"
p2panda-auth = "0.6.1"
```
