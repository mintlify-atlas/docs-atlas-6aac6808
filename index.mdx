## Key features

- **Topic-based PubSub** — nodes auto-discover peers sharing the same topic and sync all messages
- **Local-first** — data is always written locally first; sync happens when peers reconnect
- **Crash-resilient** — at-least-once delivery with acknowledgements and replay on restart
- **Group encryption** — post-compromise security and optional forward secrecy via p2panda-encryption
- **Access control** — fine-grained per-member permissions via p2panda-auth
- **Transport-agnostic** — QUIC today, designed for LoRa and BLE mesh networks in the future

<Warning>
p2panda is under active development. APIs are not yet considered stable for production use. Breaking changes may occur before v1.0.0.
