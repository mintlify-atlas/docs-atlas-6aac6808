---
title: "p2panda — high-level peer-to-peer Node API"
description: "The top-level crate that orchestrates the full p2panda stack: networking, discovery, sync, causal ordering, storage, and event streaming in one API."
---

The `p2panda` crate is the opinionated, batteries-included entry point for building decentralised peer-to-peer and local-first applications. It wires together every lower-level p2panda module — networking over QUIC/iroh, mDNS discovery, eventually-consistent log sync, causal ordering, SQLite persistence, and an event-streaming consumer model — behind a single `Node` type and two free functions: `spawn()` and `builder()`.

## Installation

```toml Cargo.toml
[dependencies]
p2panda = "0.6.1"
```

## Quick start

```rust
#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Spawn a default, in-memory node with mDNS discovery.
    let node = p2panda::spawn().await?;

    // Create a random topic and open a publish/subscribe stream.
    let topic = p2panda::Topic::random();
    let (tx, rx) = node.stream(topic).await?;

    tx.publish("Hello, Panda!".to_string()).await?;
    Ok(())
}
```

## AckPolicy enum

Every `StreamSubscription` is stateful and tracks acknowledged operations in SQLite. The policy controls when acknowledgement happens.

```rust
pub enum AckPolicy {
    /// Node assumes acknowledgment on delivery (default).
    Automatic,
    /// Each message must be explicitly acknowledged by the application.
    Explicit,
}
```

<Tip>
Switch to `AckPolicy::Explicit` and acknowledge only after successful application processing to get at-least-once delivery guarantees across crashes.
</Tip>

## StreamFrom enum

Controls the replay starting position when calling `stream_from()`.

```rust
// Resume from the stream frontier (default, used by stream()).
node.stream_from(topic, StreamFrom::Frontier).await?;

// Replay all known operations from the beginning.
node.stream_from(topic, StreamFrom::Start).await?;

// Resume from a specific cursor position.
node.stream_from(topic, StreamFrom::Cursor(my_cursor)).await?;
```

## Usage examples

### Configured node

```rust
use p2panda::RelayUrl;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let bootstrap_id =
        "1021253b4ccdfba6cac196d2b9fa6ebb605b747bcd502d6bf1d46887f04ec913".parse()?;
    let relay_url: RelayUrl = "https://my.relay.link".parse()?;

    let node = p2panda::builder()
        .database_url("db.sqlite")
        .bootstrap(bootstrap_id, relay_url.clone())
        .relay_url(relay_url)
        .spawn()
        .await?;

    Ok(())
}
```

### Typed application messages

Messages are automatically serialised to CBOR, so any `serde`-compatible type works.

```rust
use p2panda::Topic;
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize)]
enum CalendarMessage {
    Update { title: String },
    Delete,
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let node = p2panda::spawn().await?;
    let topic = Topic::random();
    let (tx, rx) = node.stream::<CalendarMessage>(topic).await?;
    Ok(())
}
```

### Awaiting publish processing

```rust
let (tx, _) = node.stream::<String>(topic).await?;

// Publish and get the operation hash immediately.
let processing = tx.publish("Hello!".into()).await?;
let hash = processing.hash();

// Optionally await the full processing result.
let result = processing.await?;
assert!(result.is_completed());
```

### System event stream

```rust
use futures_util::StreamExt;

let mut events = node.event_stream().await?;
tokio::spawn(async move {
    while let Some(event) = events.next().await {
        println!("{:?}", event);
    }
});
```

## Feature flags

| Flag | Default | Description |
|------|---------|-------------|
| `test_utils` | off | Enables test helpers on `Node` (e.g., `node.store()`). |
